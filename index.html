MultiSelectable

jQuery-plugin for making selectable lists by mouse and keyboard, configurable with options, methods and callbacks.

Download from GitHub

Options

User can select many items in a list;
Plugin lose focus when user clicks outside of the list
Plugin clear selection when user clicks outside of the list
Possibility to use keys ↓ ↑ and Home End to move cursor (focus), select several items by holding shift and select all items by pressingctrl + a;
If keyboardInput is true and focused element is last in a list, pressing key ↓ set focus to begin of the list; same with key ↑ when focus on first item of the list;
Match child elemets, that will be served as clickable area inside each item to select/unselect these items;
mouseMode
 (String) How toggle selection on an item:
mouse click select target item and clear other selection in a list; to add/remove element to selection user need holding ctrl key (file manager style);
mouse click select/unselect item (like group of checkboxes)
event (String) Which event plugin will react for:
plugin will react when user pushes down on right mouse button;
plugin will react when user clicks (press and release) on left mouse button;
plugin will select item when user pushes down mouse button on an element, that is not selected, but user must press and release mouse button to unselect selected item. It can be used, for example, when there is another plugin for dragging selected items.
filter (selector) Match child elements, that can be selected ( default = "* >" ).
wrapperClass Name of class for element which plugin attached to ( default = "j-multiSelectable" ).
focusClass Name of class on focused element ( default = "j-multiSelectable-focused" ).
selectedClass Name of class on selected element ( default = "j-multiSelectable-selected" ).
disabledClass Name of class on plugin's element which is in disabled state ( default = "j-multiSelectable-disabled" ).

Example:

$(elem).multiSelectable({ focusBlur: true, mouseMode: "click", handle: ".item-handle" });

Methods

Example of a method call: $(elem).multiSelectable("getSelected");

Turn on disabled plugin.

Detach handlers, clear data and remove plugin's classes

refresh

Refresh plugin data. That is necessary when some items in list was removed.

option

Get or set one ore more options.

Getter:

elem.multiSelectable("option", "mouseMode");

or you can get whole options object:

elem.multiSelectable("option");

Setter:

elem.multiSelectable("option", "mouseMode", "select"); elem.multiSelectable("option", { loop: true, mouseMode: "select" });

cancel

Cancel last changes in a list or prevent it, if called in beforeSelect callback. Method can be call only inside callback functions:

elem.multiSelectable({ stop: function(event, ui) { if( condition ) { this.multiSelectable("cancel"); // inside callback } } });

Select method

You can select items by plugin interface. Plugin accept elements:

elem.multiSelectable( $(".someElements") );

or selector:

elem.multiSelectable(":even");

elem.multiSelectable("li:eq(3)");

Callback functions (events)

Callback functions and arguments they receives:

event
event object from mouse or keyboard handlers.
ui
object that contains properties with DOM elements:
ui.target
link to target element, that was clicked.
ui.focus
link to focused element.
ui.items
jquery collection of changed elements – it differ from event to event.
Callback function event ui.target ui.focus ui.items
beforeSelect — calls in every work cycle before any changes will performed event if exist if exist —
select — when one or several elements is selected event target element if exist jquery collection of elements that was selected
unSelect — one or several elements has been unselected event if exist if exist jquery collection of elements that was unselected
unSelectAll — when all elements in list is unselected event if exist if exist jquery collection of elements that was unselected
focusLost — when the focus of list is lost event — Link on focused element. Link will be deleted after this callback. —
stop — calls in the end of every work cycle event if exist if exist all changed items (if they was)
create — calls after initialization — — — —
destroy — calls before plugin will be destroyed — — — —
